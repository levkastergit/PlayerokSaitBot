# Инструкция для AI-агента (Playeroksait)

Этот файл — обязательные правила при любой работе в репозитории.

---

## 1. Только то, что просят

- Делать **ровно** то, что указано в задаче. Не дополнять UI, API и поведение «от себя».
- **Запрещено** без явной просьбы:
  - подписи, пояснения, placeholder-тексты («Эта кнопка была создана…», «Здесь будет…»);
  - лишние кнопки, блоки, модалки, тосты, empty-state с маркетинговым текстом;
  - «улучшения» UX, рефакторинг соседнего кода, новые зависимости;
  - документация, тесты, коммиты — только если пользователь попросил.
- Пример: просят кнопку с текстом «Кнопка» → одна кнопка с этим текстом, без описаний под ней и без доп. действий.
- Если для корректной работы **необходимо** что-то ещё (валидация, обработка ошибки) — минимально и без лишнего текста в интерфейсе; при сомнении — уточнить у пользователя.

---

## 2. Визуал не «как у ИИ»

- Не использовать типичные AI-шаблоны: градиентные hero-блоки, «glassmorphism ради красоты», stock-иллюстрации, generic иконки (Lucide/Heroicons «на всё подряд» без причины), фиолетово-синие палитры «из коробки», карточки с одинаковыми тенями и скруглениями на каждом экране.
- Не подключать новые UI-библиотеки и наборы иконок без запроса.
- Иконки — только если нужны по смыслу; предпочитать простую графику проекта (CSS, существующие паттерны), а не декоративные SVG «для вида».

---

## 3. Единый визуальный стиль

- **Источник правды:** `frontend/src/index.css` — CSS-переменные (`--bg-*`, `--accent`, `--text-*`, `--radius-*`), классы `.app-*`, `.btn`, формы, панели, сайдбар.
- Новые стили:
  - переиспользовать существующие классы и переменные;
  - при необходимости — дописывать в `index.css` в том же духе (отступы, радиусы, цвета), а не inline-стили и не отдельные «вторые дизайн-системы» в компонентах;
  - не дублировать палитру и радиусы в JSX.
- Отдельные CSS-файлы для страницы — только если в проекте уже так сделано (например `LoginPage.css`); иначе — общий `index.css`.
- Светлая/тёмная тема: только через `[data-theme]` и переменные `:root`, не хардкодить цвета в компонентах.

---

## 4. Десктоп и мобильная версия

- Любой новый UI должен нормально работать на **ПК и на телефоне**.
- Ориентиры в проекте:
  - `.mobile-menu-toggle`, `.app-sidebar`, safe-area (`env(safe-area-inset-*)`);
  - `min-height: 100dvh` у `.app-root`;
  - существующие media queries в `index.css` — смотреть перед добавлением своих breakpoint'ов.
- Проверять: переполнение текста, кликабельные зоны (не меньше ~44px где уместно), горизонтальный скролл таблиц/форм, фиксированные ширины без `max-width: 100%`.
- Не ломать десктоп ради мобилки и наоборот; при расхождении — адаптив в `index.css`, а не два разных интерфейса.

---

## 5. Логирование по задаче — в отдельном файле

Отладочные логи **не размазывать** по бизнес-коду. Для каждой задачи/фичи — один файл, который потом легко выключить или удалить.

### Backend

- Путь: `backend/src/debug/<имя-задачи>Log.js` (папку `debug/` создавать при первой задаче).
- В начале файла — флаг:

```js
export const DEBUG_ENABLED = true // выключить задачу: false или удалить файл
```

- Все `console.log` / `console.warn` по этой задаче — **только** через функции из этого файла, с тегом для буфера логов (`GET /api/logs`):

```js
export function logTaskDebug(...args) {
  if (!DEBUG_ENABLED) return
  console.log('[task-name]', ...args)
}
```

- В handlers/services — импорт `logTaskDebug`, без сырых `console.log` по задаче.
- Продакшен-ошибки и важные `console.warn` — по необходимости, кратко, с тегом `[feature]`; временный шум — только в `debug/*Log.js`.

### Frontend

- Аналогично: `frontend/src/debug/<имя-задачи>Log.js` с `DEBUG_ENABLED` и одной точкой входа для логов; в компонентах не оставлять разбросанные `console.log` после отладки.

### После задачи

- Выключить `DEBUG_ENABLED = false` или удалить файл `debug/*Log.js`, если логи больше не нужны.
- Не коммитить секреты и токены в логах.

---

## Прочее (кратко)

- Минимальный diff; стиль кода как в соседних файлах.
- Git commit / push — только по явной просьбе пользователя.
- Секреты: не коммитить `.env`, токены, ключи.
